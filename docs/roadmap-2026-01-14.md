# Roadmap (Yandex Direct + Metrica MCP)

## Phase 0 - Foundations
- Define tool list and scope (read-only, raw exports).
- Create MCP skeleton and documentation base.

## Phase 1 - Direct (core read-only)
- Implement Direct auth + token refresh.
- Implement core tools: campaigns, ad groups, ads, keywords.
- Implement Direct reports (raw output).
- Add basic error handling and logging.

## Phase 2 - Metrica (core read-only)
- Implement Metrica counters list and counter info.
- Implement Metrica stats report tool.
- Optionally add Logs API export support.

## Phase 3 - Hardening
- Add unit tests for core tools and auth.
- Add retry/backoff and rate-limit handling.
- Improve error normalization and validation.

## Phase 4 - Docker & Ops
- Add minimal Docker image and run instructions.
- Document environment variables and setup steps.
- Add health-check script for local validation.

## Phase 5 - Optional expansions
- Add selected Direct resources from the "missing" list (if needed).
- Add data normalization layer (optional, post-MVP).

## Phase 6 - Direct write support
- Implement create/update tools for campaigns, ad groups, ads, and keywords.
- Add validation and examples for write operations.
